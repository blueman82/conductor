# Archive Directory

This directory contains historical documentation from the Conductor project development process. These documents served as working artifacts during implementation but are no longer needed in the root directory.

## Structure

### `/qa`
Quality assurance and review documentation from development cycles:
- QA findings, action items, and review reports
- Test review documentation
- Quick reference guides for QA processes

### `/delivery`
Delivery summaries and implementation completion reports:
- Feature delivery summaries
- Implementation milestone documentation
- Architecture review documents
- Documentation delivery reports

### `/task-fields`
Task field analysis and reference documentation:
- Task field dataflow analysis
- Field reference documentation
- Trace and analysis summaries

### `/cross-file`
Cross-file dependency implementation documentation:
- Implementation details and summaries
- Usage examples specific to development
- Documentation update notes

### `/parser`
Parser implementation change summaries and development notes.

## Note

For current, actively-maintained documentation, see the main `/docs` directory. These archived documents are retained for historical reference but are not actively updated.
